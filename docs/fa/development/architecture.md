# معماری

## لایه Telegram

Webhook یا Long Polling آپدیت‌های تلگرام را دریافت می‌کند و منوها، سفارش، پرداخت و نمایش سرویس را مدیریت می‌کند.

## سفارش و پرداخت

Order نیت کاربر برای خرید، تمدید یا افزایش حجم است. Payment تراکنش مربوط به آن است. تایید نهایی از مسیر `PaymentApprovalService` انجام می‌شود.

## درگاه‌های پرداخت

درگاه‌های Core داخل پروژه هستند. درگاه‌های پلاگینی بعد از validation با Plugin Bridge و adapter پرداخت load می‌شوند.

## سیستم پلاگین

پلاگین‌ها در `var/plugins/{code}` نصب می‌شوند. پشتیبانی runtime فعلی روی پلاگین‌های `payment_gateway` متمرکز است.

## پنل‌های VPN

VpnPanel و VpnInbound هدف provisioning را مشخص می‌کنند. VpnService اطلاعات سرویس ساخته‌شده، لینک اشتراک و کانفیگ‌ها را نگه می‌دارد.

## اتوماسیون

اتوماسیون مصرف را sync می‌کند، انقضا را بررسی می‌کند، اعلان می‌فرستد، سرویس‌ها را suspend می‌کند و سفارش‌های ناقص را منقضی می‌کند.

## پنل ادمین

EasyAdmin ابزار عملیاتی پروژه است و از فارسی/انگلیسی و RTL/LTR پشتیبانی می‌کند.
