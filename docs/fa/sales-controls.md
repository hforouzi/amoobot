# کنترل باز و بسته بودن فروش

از مسیر `Admin -> Settings -> Sales / فروش` می‌توانید فروش را بدون تغییر در درگاه پرداخت یا تحویل سرویس باز و بسته کنید.

## تنظیمات

- `sales.new_orders_enabled`: شروع خرید سرویس جدید را کنترل می‌کند.
- `sales.renewals_enabled`: شروع تمدید سرویس را کنترل می‌کند.
- `sales.add_traffic_enabled`: شروع خرید حجم اضافه را کنترل می‌کند.
- `sales.disabled_message`: پیامی که هنگام بسته بودن فروش به کاربر نمایش داده می‌شود.

کلیدها در پنل قابل تغییر نیستند؛ فقط مقدار تنظیمات و پیام قابل ویرایش است.

## چه چیزی مسدود می‌شود

وقتی فروش سرویس جدید غیرفعال باشد، کاربر با زدن دکمه خرید سرویس پیام غیرفعال بودن فروش را می‌بیند و پیش‌نویس یا سفارش جدید ساخته نمی‌شود.

وقتی تمدید یا خرید حجم اضافه غیرفعال باشد، شروع همان جریان مسدود می‌شود و سفارش تمدید یا خرید حجم جدید ساخته نمی‌شود.

## چه چیزی مسدود نمی‌شود

سفارش‌هایی که قبلاً به مرحله پرداخت رسیده‌اند ادامه پیدا می‌کنند. مشاهده سرویس‌های موجود، پیگیری سفارش، دریافت کانفیگ، تایید پرداخت‌های در انتظار توسط ادمین، اعلان‌ها و اتوماسیون متوقف نمی‌شوند.

## دستورات

برای ساخت تنظیمات پیش‌فرض:

```bash
php bin/console app:settings:seed
```

برای مشاهده وضعیت فروش:

```bash
php bin/console app:sales:status
```
