# مدیریت محتوای بات تلگرام

Bot Content Manager برای ویرایش متن‌های قابل مشاهده توسط کاربران تلگرام از پنل ادمین است، بدون تغییر منطق بات.

موارد قابل ویرایش:

- قالب پیام‌ها: پیام‌ها، راهنما، قوانین، درباره، اعلان‌ها و متن‌های فوتر.
- متن دکمه‌ها: متن قابل مشاهده دکمه‌های reply keyboard و inline.

موارد محافظت‌شده:

- کلید قالب و کلید دکمه شناسه داخلی هستند.
- `callback_data`، دستورها، نام stateها، وضعیت‌های سفارش/پرداخت/سرویس و اکشن‌های سیستمی قابل ویرایش نیستند.

## Fallback

متن به این ترتیب پیدا می‌شود:

1. مقدار فعال در دیتابیس برای کلید و زبان.
2. مقدار فعال در دیتابیس برای زبان پیش‌فرض.
3. ترجمه Symfony در `translations/bot.*.yaml`.
4. مقدار اضطراری داخل کد.

## متغیرها

قالب‌ها فقط placeholder ساده مثل `{{ order.trackingCode }}`، `{{ payment.amount }}`، `{{ service.expiresAt }}`، `{{ bot.brandName }}` و `{{ bot.footerText }}` پشتیبانی می‌کنند. اجرای PHP، عبارت Twig یا منطق دلخواه مجاز نیست.

## دستورها

```bash
php bin/console app:bot-content:seed --locale=fa
php bin/console app:bot-content:seed --locale=en
php bin/console app:bot-content:list
php bin/console app:bot-content:missing --locale=fa
```

گزینه `--force` رکوردهای موجود را بازنویسی می‌کند؛ برای حفظ ویرایش‌های ادمین از آن استفاده نکنید.

## متن دکمه‌ها

از مسیر **محتوای بات → متن دکمه‌ها** در ادمین فقط label قابل مشاهده را تغییر دهید. بات اکشن را با کلید داخلی دکمه و labelهای fallback تشخیص می‌دهد، بنابراین تغییر label جریان را خراب نمی‌کند. `callback_data` دکمه‌های inline ثابت می‌ماند.
